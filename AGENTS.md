# AGENTS

## 项目定位

这是一个 **ShanaEncoder 预设集合仓库**，核心资产是各目录中的编码参数与滤镜组合，而不是传统意义上的源代码工程。

目标是：
- 让不同素材（动漫、真人、3D、高低码率）快速找到可用预设；
- 在体积、画质、速度之间做可复用的折中；
- 保留历史方案，避免“改完就找不回旧策略”。

## 目录与命名约定

- `0压*`：偏复制流/低干预方案。
- `2压*`、`3压*`：不同压制强度或策略分层。
- `*SameAudio*`：保留原音轨相关策略。
- `*TransTo1080p*` / `*TransTo720p*`：带分辨率转换目标。
- `*NVENC*` / `*QSV*` / `*x264*` / `*x265*`：编码器路线区分。

新增目录时建议保持“前缀+场景+编码器/目标”的风格，便于批量筛选和脚本处理。

## 协作规则

- 不直接覆盖已稳定可用的旧预设，优先新增版本目录或新文件。
- 改参数时至少记录：适用素材、目标分辨率、质量档位（如 CRF/CQP）、关键滤镜开关。
- 需要“体积优先/速度优先/画质优先”时，拆成独立预设，不混用一个文件反复改。
- 对 CPU 与硬件编码结果分开维护，避免横向对比时混淆。
- 当前最终策略以 [PRESET_POLICY.md](D:\software\ShanaEncoder v6.0.1.4 免安装版\presets\PRESET_POLICY.md) 为准。

## 建议工作流

1. 先按素材类型选基础预设（动漫/真人/3D）。
2. 再按设备与时间预算选编码器（CPU/x265 优先画质，NVENC/QSV 优先速度）。
3. 最后调质量参数（CRF/CQP/码率上限）并做小样本验证。
4. 验证通过后再批量应用，保留测试记录以便回退。

## 当前维护重点

- 优先整理 `2压 H265 10bit x265` 相关分支为主线质量方案；
- 对硬件编码方案保留“快速可接受画质”定位；
- 为常见场景补统一命名与注释，降低后续检索成本。
